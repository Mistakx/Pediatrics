    Desenvolvimento de Aplicao de Gesto

    Introduo

    Este trabalho consiste na implementao de uma Aplicao para Gesto (AG) de empresa cujo modelo partilhado frente  (disponvel brevemente). A informao necessria para vrias actividades de gesto muitas vezes mantida em ficheiros Excel pois software comprado ou programado no facilmente altervel ou consegue acompanhar de forma eficaz a mudana do negcio e dos requisitos. Vrios processos de consolidao de informao requerem clculos e operaes manuais de copiar/colar, levando inerente redundncia de informao e possveis erros nas operaes devido ao fator humano. A AG a ser desenvolvida inclui vrios componentes abaixo descritos que pretendem automatizar e tornar mais coerentes vrias tarefas que manipulam informao importante.

    Preparao do ambiente de desenvolvimento

    A aplicao para ser desenvolvida de forma integrada no software Wordpress, tambm usado como sistema gestor de contedos online (Content Management System). O Wordpress uma aplicao desenvolvida sobre PHP+MySQL de forma a que todos os contedos de um website ficam armazenados numa BD, facilitando atualizaes, tanto dos contedos como do prprio Wordpress. Um aspeto central do desenvolvimento do TDBD a utilizao do plugin Exec-PHP que permite que posts/pginas do Wordpress executem cdigo PHP. Nomeadamente, os componentes da AG so para ser desenvolvidos em pginas gravadas numa instalao do Wordpress as quais iro conter referncias a ficheiros contendo todo o cdigo PHP e SQL necessrio. As grandes vantagens desta abordagem a rapidez e facilidade com que se organizam os vrios componentes/subcomponentes, bem como a utilizao de plugins/mecanismos/funcionalidades do prprio Wordpress para aumentar as capacidades da AG e tambm a realizao de backups/restores.

    Os alunos tero de instalar um hypervisor que possa correr uma mquina virtual que ir conter o sistema operativo Linux (distribuio Debian), incluindo o servidor web Apache com PHP, o SGBD MySQL e o Wordpress j instalados. Nomeadamente, ser necessrio realizar download desta Mquina virtual (link alternativo no onedrive da UMa) - compatvel com vrios hypervisors como: VirtualBox (Windows, Mac e Linux, clicar no link para o respetivo sistema operativo, verso mais recente), Gnome Boxes (Linux) e VMware Workstation Player (Windows e Linux), entre outros - e import-la. Para correr mquinas virtuais poder ser necessrio ativar a opo de virtualizao do CPU na BIOS do respetivo PC, no caso de Intel chama-se VT-x. preciso entrar na BIOS antes do arranque, encontrar essa configurao (pode ser necessrio ativar mostrar configuraes avanadas) e ativar.

    *** The default username and password is 'user' and 'asdfghjklzxc' ***

    Tomar nota do conjunto de 4 nmeros separados por 3 pontos (exemplo a vermelho acima) que consiste no endereo IP em que o servidor virtual est acessvel.       NOTAS IMPORTANTES

      • Para editar os ficheiros php, usar um editor que permita gravar os ficheiros com codificao utf8 e usar sempre esta codificao, seno podero ocorrer erros estranhos na execuo.
        • Recomenda-se especialmente o phpstorm (verso gratuita para estudantes aqui) dada a sua versatilidade a nvel de auto-complete, syntax highlighting e outras. Em alternativa recomenda-se tambm o Notepad++.
      • Caso o cdigo php no esteja a ser executado, ou hajam problemas de permisses, ou o erro "Internal Server Error" no acesso s pginas, executar o seguinte na consola dever resolver o problema (usar seta para cima do teclado para aceder a comandos executados recentemente na consola):
        • sudo chown -R bitnami:daemon /opt/bitnami/wordpress
      • Para evitar problemas de conexo, deve-se configurar a mquina virtual para usar NAT e no bridged (que pode dar problemas com atribuio de IPs)
      • Pode no ser possvel aceder mquina virtual ao mudar-se de rede (por exemplo de casa para UMa, de ligao de dados para wireless ou vice-versa, de wifi para cabo...)
        • Poder ser possvel ter acesso mquina configurando a rede na mquina virtual como bridged ou s com NAT ou s com ligao de dados ou s em casa por exemplo.
        • Como uma vez instalados o teclado pt, plugins e tema na mquina virtual, j no preciso esta aceder Internet, uma soluo que em princpio funciona para todos os casos configurar a rede na mquina virtual no modo host only e assim o acesso mquina virtual j ir funcionar sem conflitos. Neste modo o Edge da Microsoft pode no conseguir aceder ao IP da mquina virtual, mas o Chrome sim, ento testar com diferentes browsers. Uma forma de confirmar que a mquina virtual j est ligada e acessvel executar o comando ping xxx.yyy.ccc.ddd (substituir letras pelo IP da mquina virtual) ou tentar ligar ao servidor por ssh ou sftp.
        • Se persistirem problemas de conexo na virtualbox, algo que poder resolver a gerao de um novo mac address nas configuraes de rede
      • Caso haja perda de acesso ao wordpress mas a conexo com a mquina virtual funciona para sftp ou phpmyadmin, mesmo mudando o modo de rede, ser devido a mudana de IP ou necessidade de reiniciar a mquina virtual, sendo que ter sido atribudo um novo ip mquina. Normalmente reiniciando (comando na consola: sudo reboot) e depois executando o comando sudo /opt/bitnami/configure_app_domain --domain serverip (novo IP atribudo mquina virtual) resolve-se o problema de acesso ao wordpress.

            • echo da varivel de sesso antes e depois de a utilizar

            • certificar que NO est ser feito o unset da varivel sesso no cdigo antes de ser utilizada
